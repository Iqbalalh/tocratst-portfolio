## Overview

Nayapusaka is a centralized backend service designed to integrate and synchronize all operational data across the various divisions of Yayasan Pusaka KAI. Previously, each division maintained separate and inconsistent datasets, causing discrepancies and data fragmentation. This backend system unifies all information into a single authoritative source, ensuring accuracy, consistency, and maintainability across the entire organization.

## Key Features

- **Centralized Data Integration**: Consolidates data from all Foundation divisions into one synchronized and consistent database.
- **Scalable Backend Architecture**: Serves as the core infrastructure for all current and upcoming applications within Yayasan Pusaka KAI.
- **Secure API Services**: Provides structured RESTful APIs for internal and external applications to access unified data.
- **Cloud-Based Infrastructure**: Deployed on AWS for reliability, security, and scalability.
- **Asset Storage Management**: Uses Amazon S3 for storing documents, images, and foundation-related assets.

## Technologies Used

- **Node.js & Express.js**: For building a robust, modular, and high-performance backend service.
- **AWS EC2**: For hosting and running the backend application in a scalable cloud environment.
- **AWS RDS (PostgreSQL)**: For storing centralized, consistent, and relational data across all divisions.
- **AWS S3**: For secure storage of documents, images, and media assets.

## Outcome

Nayapusaka successfully centralized all Foundation data into a single unified system, achieving complete consistency and eliminating prior discrepancies. As a scalable backend platform, it now acts as the backbone for all Yayasan Pusaka KAI digital applications, enabling faster development, easier maintenance, and a stronger long-term infrastructure.
